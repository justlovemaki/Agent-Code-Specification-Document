## 📋 项目简介

本项目提供了一套完整的代码开发规范文档，专门用于配置 AI Coding 工具（如 Cursor、Windsurf、Claude Code、Kilo 等）的 Rules 设置。通过将这些规范集成到 AI 编程助手中，可以让 AI 生成的代码自动遵循企业级开发标准，提升代码质量和团队协作效率。

### ✨ 核心特性

- 🤖 **AI 友好** - 专为 AI Coding 工具优化的规范格式
- 📖 **开箱即用** - 可直接复制到 AI 工具的 Rules 配置中
- 🎯 **按需启用** - 根据项目技术栈选择性启用规范
- 🔧 **多技术栈** - 支持 Java、Next.js、TypeScript 等主流技术
- 📚 **持续更新** - 跟随技术发展和最佳实践不断完善

## 🚀 快速开始

### 使用方式

#### 1️⃣ 在 Cursor 中使用

1. 打开 Cursor 设置（`Ctrl/Cmd + ,`）
2. 找到 "Rules for AI" 或 "Cursor Rules" 设置项
3. 根据项目需求，复制对应的规范文档内容
4. 粘贴到 Rules 配置区域
5. 保存设置，AI 将自动遵循这些规范

#### 2️⃣ 在 Windsurf 中使用

1. 打开 Windsurf 的 Rules 配置文件（通常是 `.windsurfrules` 或项目根目录的 `rules.md`）
2. 选择需要的规范文档
3. 复制内容到配置文件中
4. 保存后 AI 将按照规范生成代码

#### 3️⃣ 在其他 AI Coding 工具中使用

大多数 AI 编程助手都支持自定义规则配置，查找以下设置项：
- Custom Instructions
- AI Rules
- Code Guidelines
- Project Context

将对应的规范文档内容复制到配置中即可。

## ❓ 常见问题

### Q: 如何选择合适的规范？
**A:** 根据项目使用的技术栈选择对应的规范文档。如果是全栈项目，可以同时启用前后端规范。

### Q: 规范内容太长，会影响 AI 性能吗？
**A:** 现代 AI 工具都能很好地处理长文本规范。如果担心性能，可以只启用最核心的部分。

### Q: 可以修改规范内容吗？
**A:** 完全可以！建议根据团队实际情况调整规范，形成适合自己团队的版本。

### Q: AI 不遵循规范怎么办？
**A:** 在提示词中明确强调要遵循规范，或者在 Rules 配置中添加更明确的指令。

### Q: 如何确保团队成员都使用相同的规范？
**A:** 将 Rules 配置文件（如 `.cursorrules` 或 `rules.md`）提交到代码仓库，团队成员克隆后自动生效。
